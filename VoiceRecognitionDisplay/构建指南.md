# 多平台构建指南

## 快速构建

### Android APK

**Windows:**
```cmd
build-android.bat
```

**macOS:**
```bash
./build-android.sh
```

**输出位置:** `VoiceRecognitionDisplay.Android/bin/Release/net8.0-android34.0/com.voicerecognition.display-Signed.apk`

---

### Windows 桌面版

```cmd
build-windows.bat
```

**输出位置:** `VoiceRecognitionDisplay/bin/Release/net8.0/win-x64/publish/VoiceRecognitionDisplay.exe`

**特点:**
- 独立运行（无需安装 .NET）
- 单个可执行文件
- 大小约 80-100 MB

---

### macOS 桌面版

```bash
./build-macos.sh
```

**输出位置:** `VoiceRecognitionDisplay/bin/Release/net8.0/osx-x64/publish/VoiceRecognitionDisplay`

**运行方式:**
```bash
./VoiceRecognitionDisplay/bin/Release/net8.0/osx-x64/publish/VoiceRecognitionDisplay
```

**特点:**
- 独立运行（无需安装 .NET）
- 单个可执行文件
- 大小约 80-100 MB

---

## 系统要求

### Android
- 最低版本: Android 5.0 (API 21)
- 目标版本: Android 14 (API 34)
- 需要权限: 网络访问、WiFi 状态、唤醒锁

### Windows
- 最低版本: Windows 10 (1809) 或更高
- 架构: 64位 (x64)

### macOS
- 最低版本: macOS 10.15 (Catalina) 或更高
- 架构: Intel (x64) 或 Apple Silicon (arm64)

---

## Apple Silicon (M1/M2) Mac

如果你使用 M1/M2 Mac，可以构建原生版本：

```bash
cd VoiceRecognitionDisplay/VoiceRecognitionDisplay
dotnet publish -c Release -r osx-arm64 --self-contained true -p:PublishSingleFile=true
```

## 分发说明

### Android
将 APK 文件传输到设备：
- USB 数据线
- 云存储（百度网盘、微云等）
- 邮件
- 在设备上启用"允许安装未知来源应用"

### Windows
直接分发 `.exe` 文件即可运行。

### macOS
分发可执行文件。首次运行时：
1. 右键点击 → 打开
2. 或在"系统偏好设置" → "安全性与隐私"中允许

## 常见问题

### Android 构建失败

1. 确认已安装 Android workload:
   ```cmd
   dotnet workload install android
   ```

2. 运行 SDK 检测:
   ```cmd
   find-android-sdk.bat
   ```

### Windows 构建失败

1. 检查 .NET 版本:
   ```cmd
   dotnet --version
   ```
   应该显示 8.0.x

2. 清理并重新构建:
   ```cmd
   dotnet clean
   dotnet build
   ```

### macOS 构建失败

1. 确保脚本有执行权限:
   ```bash
   chmod +x build-macos.sh
   ```

2. 检查 .NET SDK:
   ```bash
   dotnet --version
   ```
