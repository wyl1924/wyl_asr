## 日志分析重点

查看日志文件 `android_window_debug.log`，重点关注：

### 1. 窗口高度设置
```
目标窗口高度: [应该是 720px 左右] (240dp)
```

### 2. 修改后的窗口属性
```
修改后窗口属性:
  Height: [应该等于目标高度]
  Gravity: Bottom
  NotFocusable: True
```

### 3. 实际渲染尺寸
```
DecorView 尺寸:
  Height: [这个值很关键！]
```

**关键问题：**
- 如果 DecorView.Height 等于目标高度（约 720px）→ 设置成功 ✅
- 如果 DecorView.Height 等于屏幕高度（约 2400px）→ 设置被覆盖 ❌

### 4. OnResume 时的状态
```
OnResume 时窗口属性:
  Height: [检查是否变化]
```

如果这里的高度变回全屏，说明 Avalonia 在渲染时覆盖了我们的设置。

## 可能的结果

### 情况 A：窗口高度设置成功
如果日志显示 DecorView.Height 约等于 720px，但屏幕上仍然看到白色区域，可能是：
- XAML 布局问题
- 透明度设置问题

### 情况 B：窗口高度被覆盖
如果日志显示 DecorView.Height 仍然是全屏高度，说明 Avalonia 覆盖了我们的设置。

**解决方案：**
需要在 `OnResume()` 中重新强制设置窗口高度，或者考虑使用 Android 原生悬浮窗口服务（Service + WindowManager）。
