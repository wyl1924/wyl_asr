# Android APK 构建 - 快速开始

## 问题：构建脚本找不到 Android SDK

你已经安装了 Android SDK，但构建脚本没有找到它。

## 解决方案（3步）

### 第1步：查找 SDK 位置

运行：
```cmd
find-android-sdk.bat
```

这会显示你的 Android SDK 安装在哪里。

### 第2步：设置环境变量

运行：
```cmd
set-android-env.bat
```

这会自动设置 `ANDROID_HOME` 和 `ANDROID_SDK_ROOT` 环境变量。

### 第3步：重启命令提示符并构建

1. **关闭当前命令提示符**
2. **打开新的命令提示符**（这样才能加载新的环境变量）
3. 运行：
   ```cmd
   cd VoiceRecognitionDisplay
   build-android.bat
   ```

## 如果 SDK 在特殊位置

如果你的 Android SDK 安装在非标准位置，手动设置环境变量：

```cmd
setx ANDROID_HOME "你的SDK完整路径"
setx ANDROID_SDK_ROOT "你的SDK完整路径"
```

例如：
```cmd
setx ANDROID_HOME "D:\Android\Sdk"
setx ANDROID_SDK_ROOT "D:\Android\Sdk"
```

然后重启命令提示符，再运行 `build-android.bat`。

## 验证 SDK 路径

你的 Android SDK 目录应该包含这些文件夹：
- `platform-tools` (包含 adb.exe)
- `platforms` (包含 android-34 等)
- `build-tools` (包含 34.0.0 等版本)

## 构建成功后

APK 文件位置：
```
VoiceRecognitionDisplay.Android\bin\Release\net8.0-android34.0\com.voicerecognition.display-Signed.apk
```

将这个 APK 文件传输到 Android 设备并安装。

## 常见问题

### Q: 运行 `set-android-env.bat` 后还是找不到 SDK？

A: 必须关闭当前命令提示符，打开新的命令提示符。环境变量只在新的会话中生效。

### Q: 如何检查环境变量是否设置成功？

A: 在新的命令提示符中运行：
```cmd
echo %ANDROID_HOME%
```
应该显示你的 SDK 路径，而不是 `%ANDROID_HOME%`。

### Q: 构建时出现中文乱码？

A: 脚本已经设置了 UTF-8 编码 (`chcp 65001`)，应该不会有乱码。

### Q: 构建成功但找不到 APK？

A: 检查构建输出中是否有错误。APK 应该在：
```
VoiceRecognitionDisplay.Android\bin\Release\net8.0-android34.0\
```
