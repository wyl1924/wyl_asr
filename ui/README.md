# WYL ASR 前端设置页面

## 功能概述

本前端应用新增了一个设置页面，用于配置和管理 FunASR WebSocket 实时语音识别服务器的启动参数。通过可视化界面，用户可以轻松配置各种参数并启动后端服务。

## 主要功能

### 🔧 设置页面功能

1. **服务器状态监控**
   - 实时显示后端服务运行状态
   - 显示服务进程ID和启动时间
   - 服务运行时长统计

2. **服务器控制**
   - 启动服务器
   - 停止服务器
   - 重启服务器

3. **参数配置**
   - **网络配置**: IP地址、端口、SSL证书
   - **模型配置**: ASR模型类型、VAD模型、标点恢复模型
   - **2Pass模式**: 在线+离线双通道识别配置
   - **硬件配置**: CPU/GPU设备选择、线程数配置
   - **说话人识别**: 统一开关控制说话人识别和分离功能
   - **翻译功能**: 中英翻译模型配置

4. **配置管理**
   - 保存配置到本地存储
   - 加载已保存的配置
   - 重置为默认配置
   - 保存并启动服务

## 使用方法

### 1. 启动前端开发服务器

```bash
cd ui
npm install
npm run dev
```

### 2. 访问设置页面

1. 打开浏览器访问 `http://localhost:5173`
2. 在首页点击「系统设置」按钮
3. 或直接访问 `http://localhost:5173/settings`

### 3. 配置和启动服务

1. **配置参数**：
   - 根据需要调整网络、模型、硬件等配置
   - 可以选择不同的模型类型（SenseVoice或Paraformer）
   - 配置2Pass模式以获得最佳识别效果

2. **保存配置**：
   - 点击「保存配置」按钮将配置保存到本地浏览器
   - 下次访问时会自动加载保存的配置

3. **生成启动命令**：
   - 点击「生成启动命令」按钮获取完整的启动命令
   - 或点击「保存并生成命令」一键保存配置并生成命令
   - 命令会自动复制到剪贴板

4. **手动启动服务**：
   - 在终端中切换到项目根目录
   - 粘贴并执行生成的启动命令
   - 例如：`conda activate funasr-ws && python main.py --host 0.0.0.0 --port 10095 ...`

## 配置说明

### 网络配置
- **服务器IP地址**: 默认 `0.0.0.0`（监听所有接口）
- **WebSocket端口**: 默认 `10095`
- **API端口**: 默认 `8080`
- **SSL证书**: 可选，用于HTTPS连接

### 模型配置
- **SenseVoice**: 多模态模型，支持语言检测、情感识别
- **Paraformer**: 传统高精度识别模型
- **VAD模型**: 语音活动检测
- **标点恢复**: 自动添加标点符号

### 2Pass模式
- **启用**: 结合在线和离线识别的优势
- **全局Beam**: 解码搜索参数
- **声学模型缩放**: 模型权重调整

### 硬件配置
- **CPU**: 适用于所有环境
- **CUDA**: NVIDIA GPU加速
- **MPS**: Apple Silicon GPU加速

## 工作原理

设置页面通过以下方式工作：

1. **参数配置**: 用户在界面中配置各种启动参数
2. **命令生成**: 根据配置生成完整的Python启动命令
3. **本地存储**: 配置保存在浏览器的localStorage中
4. **手动执行**: 用户复制命令到终端手动执行

## 注意事项

1. **环境要求**：
   - 确保已安装Python环境和相关依赖
   - 激活正确的conda环境：`conda activate funasr-ws`

2. **端口冲突**：
   - 如果端口被占用，设置页面会提示错误
   - 可以在配置中修改端口号

3. **模型下载**：
   - 首次使用时会自动下载模型文件
   - 确保网络连接正常

4. **权限问题**：
   - 在某些系统上可能需要管理员权限
   - 确保有足够的磁盘空间存储模型

## 故障排除

### 服务启动失败
1. 检查Python环境是否正确
2. 确认所有依赖已安装
3. 查看控制台错误信息
4. 检查端口是否被占用

### 前端无法连接后端
1. 确认后端服务已启动
2. 检查防火墙设置
3. 验证IP地址和端口配置

### 模型加载失败
1. 检查网络连接
2. 确认模型路径正确
3. 验证磁盘空间充足
